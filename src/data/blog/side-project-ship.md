<p>Developer တော်တော်များများ side project တွေ စတင်ပြီး မပြီးဘဲ ထားခဲ့ကြတယ်။ ဒါဟာ laziness မဟုတ်ဘဲ wrong approach ကြောင့်ပါ။</p>
<h2>Perfectionism ကို ဖြတ်တောက်ပါ</h2>
<p>"Perfect ဖြစ်မှ launch လုပ်မယ်" ဆိုတဲ့ thinking ကို ဖြတ်ပါ။ V1 မှာ core feature တစ်ခုပဲ ထည့်ပါ။ ကျန်တာ V2 မှာ ထည့်ပါ။</p>
<h2>Time-box လုပ်ပါ</h2>
<p>Project တစ်ခုအတွက် 2-week deadline ပေးပါ။ Deadline မရောက်ခင် ship လုပ်ပါ — complete မဖြစ်ရင်တောင်မှ။</p>
<h2>Public Accountability</h2>
<p>Twitter/X မှာ "Building in public" လုပ်ပါ။ Audience ရှိရင် quit လုပ်ဖို့ ခက်လာတယ်။</p>
<h2>Done > Perfect</h2>
<p>Shipped ပေမဲ့ imperfect ဖြစ်တဲ့ project ဟာ locally only run တဲ့ perfect project ထက် တန်ဖိုးများစွာ ရှိပါတယ်။ Users တွေ feedback ပေးမှ ဘာ build ရမလဲ သိလာမှာပါ။</p>
